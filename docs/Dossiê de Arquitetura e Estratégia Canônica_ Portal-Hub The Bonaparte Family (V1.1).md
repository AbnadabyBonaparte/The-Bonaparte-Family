## 1. Resumo Executivo: A Visão 1000/10000

O **Portal-Hub The Bonaparte Family** é a síntese arquitetônica do projeto de vida de Abnadaby Bonaparte. Não se trata de um website comum, mas sim de um **Sistema de Informação Canônico** projetado para unificar as múltiplas camadas da Casa Bonaparte: a **Família** (humana, nômade, consciente), o **Império** (ALSHAM Global Commerce, tecnologia, agentes) e o **Legado** (filosofia, obra, futuro).

O objetivo é criar um portal de classe mundial que reflita a complexidade e a profundidade do projeto, sendo acessível e inspirador para o público (PT/EN) e, ao mesmo tempo, robusto e escalável para a arquitetura empresarial (ALSHAM). A identidade visual canônica escolhida é o **Híbrido (C)**, que harmoniza a estética documental e a funcionalidade tecnológica.

| Critério | Descrição Canônica |
| :--- | :--- |
| **Nome Global** | The Bonaparte Family |
| **Identidade Central** | Família + Império + Tecnologia + Legado |
| **Padrão de Qualidade** | Obra-Prima 10/10 - 1000/10000 |
| **Estilo Visual Canônico** | Híbrido (Documentário + Tech) |
| **Idiomas de Lançamento** | Português (PT) e Inglês (EN) |
| **Escopo do MVP** | Portal-Hub completo (13 rotas) sem área privada de login |

## 2. Arquitetura Técnica Canônica (Stack Supremo)

A arquitetura técnica é baseada nos canônicos já estabelecidos nos repositórios da Casa Bonaparte, garantindo **coerência, escalabilidade e performance** de nível empresarial.

| Componente | Tecnologia Canônica | Justificativa Estratégica |
| :--- | :--- | :--- |
| **Frontend/Framework** | Next.js 14+ (App Router) | Server Components, Streaming, SEO nativo e performance insana, alinhado com `ALSHAM-360-PRIMA`. |
| **Linguagem** | TypeScript Strict | Tipagem rigorosa para zero *runtime errors* e consistência entre os sistemas. |
| **Estilização/UI** | Tailwind CSS + shadcn/ui | Design com canônicos visuais, reutilização de componentes e aderência ao *design system* da Casa. |
| **Backend/DB** | Supabase (Postgres, Auth, Storage, Realtime) | Solução *full-stack* escalável, alinhada com o uso atual em múltiplos repositórios. |
| **Automação/Agentes** | n8n/Make + LangChain | Para o portal ser "vivo": atualizações automáticas de mapa, timeline e integração com agentes ALSHAM. |
| **Deploy** | Vercel | *Zero config*, *edge functions* e *preview branches* para desenvolvimento ágil e escalável. |
| **Mapa Interativo** | Leaflet ou Mapbox | Essencial para a seção `Expedition`, com integração *realtime* via Supabase. |

## 4. Design System Híbrido Canônico (Estilo C)

O estilo visual **Híbrido** (Documentary + Tech) é a escolha canônica para refletir a dualidade da Casa Bonaparte: a **calidez humana** da família e a **precisão fria** da arquitetura de sistemas.

| Elemento | Especificação Canônica | Implicação Visual |
| :--- | :--- | :--- |
| **Fundo** | Linen/Sand (Claro) | Base de documentário, calidez e alta legibilidade. |
| **Acentos** | Olive/Verde-Musgo | Conexão com a natureza, a estrada e a vida no sítio. |
| **Tipografia** | Serif (Títulos) + Sans (UI/Corpo) | Elegância clássica para o Legado, clareza moderna para a Tecnologia. |
| **Recursos** | shadcn/ui + Toggle Dark Mode | Componentes profissionais e a opção de alternar para o modo *Tech-Imperial* (Escuro). |
| **Hero Section** | Split Layout (50/50) | Lado esquerdo com texto (`The Bonaparte Family`), lado direito com colagem de *placeholders* de fotos reais (estrada, ferramentas, mapa, notebook). |

## 5. Governança e Regras Sagradas (Leis Invioláveis)

A integridade do Portal-Hub é mantida por um conjunto de regras de governança que garantem a autenticidade e o alinhamento com a visão de longo prazo.

> **Regra Suprema:** Nada entra no portal sem responder: **isso fortalece a vida real + a obra + o legado?**

| Regra Sagrada | Descrição e Aplicação |
| :--- | :--- |
| **Vida Real, Não Performance** | Curadoria rigorosa: exposição com propósito (documentário), não para *performance* de redes sociais. |
| **Prova Acima de Promessa** | O mapa, a *timeline*, o diário e a construção do Alfredo são a prova de vida real e o motor da narrativa. |
| **Design Canônico** | Consistência visual e técnica (shadcn/ui, TS strict) é mais importante do que improviso ou modismos. |
| **ALSHAM Não Engole o Humano** | A empresa é o motor econômico da liberdade, mas a família e o legado são o centro da narrativa. |
| **Multilíngue (PT/EN)** | O portal deve ser global desde o lançamento, com tradução de conteúdo canônico. |
| **Legado Protegido** | Embora a área privada não esteja no MVP, a arquitetura deve prever a **Legacy Vault** para o futuro. |
